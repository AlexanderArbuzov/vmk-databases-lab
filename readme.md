## Databases, Lab 
### UNN, Applied Informatics (3rd year)
<br><b>Предметная область</b><br><br>
Вы работаете для мелкооптового магазина. У Вас есть список видов товаров (артикул (пятизначное число) - должен являться первичным ключом), наименование
товаров, цена за единицу товара (не может быть отрицательной). Обычно покупают сразу много видов товара большими количествами.
Вам необходимо хранить общую сумму покупки, а так же сколько товара какого вида содержится в этой покупке.<br><br>
Артикулы могут обновляться, необходимо чтобы при обновлении артикула в покупках ссылки также обновлялись. Пока в какой-либо покупке есть ссылка на вид товара этот вид товара удален быть не может. Покупки могут возвращаться, в этом случае все данные по покупке также должны быть удалены.
<br><br><b>Задание</b><br>
<ul>
<li>Нарисуйте схему таблиц (можно на бумаге) и покажите преподавателю</li>
<li>Заполните тестовыми данными (3 записи в каждой таблице). Покупки должны
  быть заполнены с использованием последовательностей</li>
<li>Попробуйте обновление артикула, удаление покупки, удаление вида товара - на
  которые есть ссылки, убедитесь, что требования ТЗ в части удаления и обновления
  выполняются</li>
<li>Напишите триггер обновления суммы покупки при добавлении в нее товара</li>
</ul>

(Рабочая программа дисциплины «Базы данных», Н.Новгород, 2020)
